# 移动端分布式 Harness Agent 方案评估

> 多手机分布式 AI Agent 架构可行性分析

## 1. 方案描述

### 方案一：分布式计算

```
┌─────────────┐     任务分发      ┌─────────────┐
│  主手机     │ ───────────────► │ 手机 A     │
│ (Master)    │                  │ (Worker 1) │
│            │                  ├─────────────┤
│            │                  │ 手机 B     │
│            │                  │ (Worker 2) │
│            │                  ├─────────────┤
│            └───────────────► │ 手机 C     │
│            (并行/队列)        │ (Worker 3) │
└─────────────┘                  └─────────────┘

- 所有手机运行 Hermes/ACPX Agent
- 主手机负责任务分发与结果汇总
- 从手机执行具体计算任务
```

### 方案二：多 Agent 协作

```
┌─────────────┐    协调         ┌─────────────┐
│  主手机    │ ◄────────────► │ 研究 Agent │
│ (Coordinator)              │ (手机A)    │
│                           ├─────────────┤
│ 任务路由                   │ 代码 Agent │
│ + 结果聚合                 │ (手机B)    │
│                           ├─────────────┤
│                           │ 搜索 Agent │
│                           │ (手机C)    │
└─────────────┘               └─────────────┘

- 每个手机有特定角色/技能
- 主手机充当中间协调层
- 类似 Multi-Agent 系统
```

## 2. 可行性分析

### ✅ 优势

| 方面 | 说明 |
|------|------|
| **并行能力** | 多手机可并行处理任务 |
| **容错性** | 单机故障不影响整体 |
| **扩展性** | 按需增加节点 |
| **成本低** | 现有手机无需采购硬件 |
| **灵活部署** | 角色可动态调整 |

### ⚠️ 挑战

| 挑战 | 说明 | 严重程度 |
|------|------|---------|
| **网络延迟** | 需稳定局域网或 VPN | ⭐⭐⭐ |
| **通信协议** | 需定义 Agent 间协议 | ⭐⭐⭐ |
| **状态同步** | 多机状态一致性 | ⭐⭐⭐ |
| **任务分割** | 如何拆分任务 | ⭐⭐ |
| **安全问题** | 内网暴露/权限 | ⭐⭐ |
| **Termux 限制** | 无 root/Docker | ⭐⭐ |

### ❌ 技术限制

| 限制 | 影响 |
|------|------|
| **无 root** | 无法运行系统级服务 |
| **无 Docker** | 无法容器化 |
| **CPU 限制** | 手机 CPU 弱于服务器 |
| **内存限制** | 8GB 内存上线 |
| **存储限制** | 空间有限 |
| **电量** | 长时间运行发热 |

## 3. 技术架构

### 3.1 通信层

```bash
# 方案A: 局域网 WebSocket
# 主手机: ws://192.168.1.x:8765

# 方案B: MQTT mosquitto
# 主手机运行 MQTT broker

# 方案C: HTTP REST
# 简单轮询，延迟高
```

### 3.2 协议设计

```
┌─────────────────────────────────────┐
│           ACP 扩展                  │
├─────────────────────────────────────┤
│ task/distribute  -> 分发任务          │
│ task/report    <-  返回结果         │
│ status/heartbeat  -> 心跳保活      │
│ session/proxy   -> 代理会话        │
└─���───────────────────────────────────┘
```

### 3.3 角色定义

```yaml
# 手机配置示例
device_id: "phone-a"
role: "researcher"      # 研究/搜索
capabilities:
  - web_search
  - read_docs
  - summarize

device_id: "phone-b"  
role: "coder"
capabilities:
  - write_code
  - debug
  - test

role: "coordinator"     # 主手机
device_id: "phone-main"
capabilities:
  - task_dispatch
  - result_aggregate
  - routing
```

## 5. 性能评估

### 理论性能

| 节点数 | 相对单手机 | 场景 |
|--------|----------|------|
| 1 | 1x | 基准 |
| 2 | 1.8x | 并行任务 |
| 3 | 2.5x | 分工协作 |
| 5 | 3.5x | 复杂多任务 |

**注意**: 受网络延迟和协调开销，实际效率低于线性。

|------|------|
