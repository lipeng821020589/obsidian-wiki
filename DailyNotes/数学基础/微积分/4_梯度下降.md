## 基本思想

梯度下降法是一种**一阶优化算法**，用于最小化目标函数。

### 更新规则
$$\mathbf{w}_{t+1} = \mathbf{w}_t - \eta \nabla \mathcal{L}(\mathbf{w}_t)$$

其中 $\eta > 0$ 是学习率（step size）。

### 几何解释
- 梯度 $\nabla \mathcal{L}$ 指向函数增长最快的方向
- 负梯度 $-\nabla \mathcal{L}$ 指向函数**下降最快**的方向
- 学习率 $\eta$ 控制每一步的步长

---

## 收敛性分析

### 收敛条件
若目标函数 $\mathcal{L}$ 是**凸函数**且满足Lipschitz连续：
$$|\nabla \mathcal{L}(\mathbf{x}) - \nabla \mathcal{L}(\mathbf{y})| \leq L |\mathbf{x} - \mathbf{y}|$$

### 收敛率
$$\mathcal{L}(\mathbf{w}_t) - \mathcal{L}(\mathbf{w}^*) \leq \frac{|\mathbf{w}_0 - \mathbf{w}^*|^2}{2\eta t}$$

对于强凸函数，收敛率提升到 $O(\exp(-t))$。

---

## 梯度下降的变体

### 1. 批量梯度下降 (Batch GD)
使用**全部数据**计算梯度：
$$\nabla \mathcal{L}(\mathbf{w}) = \frac{1}{N}\sum_{i=1}^N \nabla \mathcal{L}_i(\mathbf{w})$$

- **优点**：稳定收敛，方向准确
- **缺点**：计算量大，内存要求高
- **场景**：小数据集

### 2. 随机梯度下降 (SGD)
使用**单个样本**或**小批量**：
$$\mathbf{w}_{t+1} = \mathbf{w}_t - \eta \nabla \mathcal{L}_{i_t}(\mathbf{w}_t)$$

- **优点**：快，内存需求低，能逃离局部极小
- **缺点**：梯度有噪声，收敛不稳定
- **场景**：大数据集

### 3. Mini-batch SGD
使用**固定大小**的批次：
$$\mathbf{w}_{t+1} = \mathbf{w}_t - \eta \frac{1}{B}\sum_{i \in \mathcal{B}_t} \nabla \mathcal{L}_i(\mathbf{w}_t)$$

- **优点**：平衡效率和稳定性
- **缺点**：需要调优batch size
- **场景**：深度学习标准做法

---

## 学习率调度 (Learning Rate Scheduling)

固定学习率可能难以收敛。常用调度策略：

### 1. Step Decay
每隔固定epoch降低学习率：
```python
if epoch % 30 == 0:
    lr *= 0.1
```

### 2. Cosine Annealing
$$\eta_t = \eta_{\min} + \frac{1}{2}(\eta_{\max} - \eta_{\min})\left(1 + \cos\left(\frac{t\pi}{T}\right)\right)$$

### 3. Warmup + Decay
先预热后衰减（Transformer常用）：
```python
# Warmup阶段
lr = base_lr * (step / warmup_steps)

# 之后按cosine衰减
lr = base_lr * 0.5 * (1 + cos((step - warmup_steps) / total_steps * pi))
```

### 4. Reduce on Plateau
验证集loss不再下降时降低学习率：
```python
# ▶ 学习率调度示例
import torch
import torch.optim as optim

model = torch.nn.Linear(10, 2)
optimizer = optim.AdamW(model.parameters(), lr=1e-3)
scheduler = optim.lr_scheduler.ReduceLROnPlateau(
    optimizer, mode='min', factor=0.5, patience=5
)

# 模拟训练
for epoch in range(20):
    loss = torch.rand(1)
    scheduler.step(loss)
    if epoch % 5 == 0:
        print(f"Epoch {epoch}: lr={optimizer.param_groups[0]['lr']:.6f}")
```

## 动量法 (Momentum)

### 问题
SGD在峡谷（curved valleys）中震荡，难以快速下降。

### 动量更新
$$m_t = \beta m_{t-1} + (1 - \beta) \nabla \mathcal{L}(\mathbf{w}_t)$$
$$\mathbf{w}_{t+1} = \mathbf{w}_t - \eta m_t$$

### 物理理解
$m_t$ 累积历史梯度，类似物理中的动量，帮助穿越平坦方向。

### 参数选择
- $\beta$ 通常取 0.9
- 相当于使用最近 $1/(1-\beta) = 10$ 步梯度的指数加权平均

### Nesterov动量
先看未来位置：
$$m_t = \beta m_{t-1} + (1-\beta) \nabla \mathcal{L}(\mathbf{w}_t - \eta \beta m_{t-1})$$
$$\mathbf{w}_{t+1} = \mathbf{w}_t - \eta m_t$$

收敛更快，理论上优于标准动量。

---

## AdaGrad

### 自适应学习率
$$r_t = r_{t-1} + (\nabla \mathcal{L})^2$$
$$\mathbf{w}_{t+1} = \mathbf{w}_t - \frac{\eta}{\sqrt{r_t} + \epsilon} \nabla \mathcal{L}(\mathbf{w}_t)$$

### 特点
- 对稀疏特征友好（稀疏梯度累积少，学习率保持较大）
- 缺点：$r_t$ 持续增长，学习率会越来越小

---

## RMSProp

### 改进AdaGrad
$$r_t = \beta r_{t-1} + (1-\beta)(\nabla \mathcal{L})^2$$
$$\mathbf{w}_{t+1} = \mathbf{w}_t - \frac{\eta}{\sqrt{r_t} + \epsilon} \nabla \mathcal{L}(\mathbf{w}_t)$$

### 特点
- 使用指数加权平均，解决了学习率衰减过快的问题
- 适合非稳态问题
- 默认 $\beta = 0.9$

---

## Adam (Adaptive Moment Estimation)

Adam结合了动量和RMSProp的思想。

### 算法
```
初始化: m_0 = 0, v_0 = 0, t = 0
对于每个step:
    t += 1
    g_t = gradient
    
    # 动量（类似梯度的一阶矩估计）
    m_t = beta1 * m_{t-1} + (1 - beta1) * g_t
    
    # RMSProp（类似梯度的二阶矩估计）
    v_t = beta2 * v_{t-1} + (1 - beta2) * g_t^2
    
    # 修正偏差（bias correction）
    m_hat = m_t / (1 - beta1^t)
    v_hat = v_t / (1 - beta2^t)
    
    # 更新
    w_{t+1} = w_t - lr * m_hat / (sqrt(v_hat) + eps)
```

### 默认参数
- $\beta_1 = 0.9$（一阶矩）
- $\beta_2 = 0.999$（二阶矩）
- $\epsilon = 10^{-8}$
- $\text{lr} \in [10^{-4}, 10^{-3}]$

### 优点
- 几乎不需要调参
- 对超参数选择鲁棒
- 适合大多数问题

---

## 优化算法对比

| 算法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| SGD | 可解释，稳定 | 收敛慢，需调学习率 | 经典研究 |
| Momentum | 加速收敛 | 需调 $\beta$ | 一般场景 |
| AdaGrad | 自适应，不需学习率 | 学习率单调下降 | 稀疏数据 |
| RMSProp | 自适应，处理非稳态 | 依赖超参数 | RNNs |
| Adam | 鲁棒，自动调整 | 可能泛化差 | 深度学习默认 |

### 实践建议
- **首选Adam**（快，省心）
- **追求泛化**：SGD + Momentum + Cosine Annealing（需要仔细调参）
- **训练大模型**：AdamW（带权重衰减的Adam）

## AdamW (Weight Decay Adam)

### L2正则 vs 权重衰减
- L2正则：在梯度上加 $\lambda \mathbf{w}$（会受学习率缩放影响）
- 权重衰减：直接在参数上衰减 $\mathbf{w} \leftarrow (1 - \eta\lambda)\mathbf{w}$

AdamW正确实现了权重衰减，而非L2正则。

```python
# ▶ AdamW vs Adam (L2正则)
import torch
import torch.optim as optim

model = torch.nn.Linear(10, 2)

# AdamW (正确实现)
optimizer_wd = optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)

# Adam with L2 (不等价！)
optimizer_l2 = optim.Adam(model.parameters(), lr=1e-3, weight_decay=0.01)

# AdamW: 权重直接衰减 (1 - lr*wd)
# Adam L2: 梯度上加上 wd * w (受学习率影响)
```

### 效果
- 权重分布更合理
- 学习率和权重衰减可独立调优

## 梯度裁剪 (Gradient Clipping)

### 梯度爆炸问题
RNNs和Transformers中梯度可能爆炸。

### 方法1：按值裁剪
```python
# ▶ 梯度裁剪示例
import torch.nn.utils as utils

model = torch.nn.Linear(10, 2)
x = torch.randn(5, 10)
loss = model(x).sum()
loss.backward()

print(f"裁剪前梯度范数: {utils.clip_grad_norm_(model.parameters(), max_norm=1.0):.4f}")
print(f"裁剪后: {model.weight.grad.norm().item():.4f}")
```

### 数学
$$g = \begin{cases} g \cdot \frac{\max\_norm}{\|g\|} & \|g\| > \max\_norm \\ g & \text{otherwise} \end{cases}$$

### 在Transformer中的应用
BERT、GPT等通常设置 `max_norm = 1.0`。
